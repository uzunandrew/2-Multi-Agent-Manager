# Инструкция для vision-модели: структурированное описание чертежей слаботочных систем

Вы получаете PNG-изображение из проектной документации слаботочных систем (раздел СС). Ваша задача — описать его СТРУКТУРИРОВАННО, чтобы техническое содержание можно было восстановить по описанию без просмотра изображения.

## Область применения и ограничения

**7 типов чертежей слаботочных систем (определите тип и примените соответствующий формат):**

| Тип | Доля | Что это |
|-----|------|---------|
| Структурные схемы | 25% | Шлейфы АПС, топология СКУД, дерево СОТ, сбор АСКУЭ |
| Планы этажей с устройствами | 30% | Размещение извещателей/камер/считывателей/оповещателей на планах этажей |
| Электрические принципиальные схемы | 10% | Подключение ППКП, разводка контроллера, цепи питания |
| Схемы подключения | 10% | Разводка клеммных колодок, подключение устройств к контроллеру |
| Планы кабельных трасс | 10% | Трассы слаботочных лотков, схемы стояков |
| Таблицы и условные обозначения | 10% | Спецификации, кабельные журналы, условные обозначения |
| Функциональные схемы автоматизации | 5% | Логика датчик-контроллер-исполнительный механизм, схемы АСУД |

**Что НЕ описывать (эти данные уже есть в document.md):**
- Основные надписи, титульные надписи (номер листа, шифр, организация)
- Таблицы изменений
- Номера листов и их наименования
- Ведомости чертежей

**Для каждого типа изображения** определите тип и примените соответствующий формат описания ниже.

## Принцип чтения

Слаботочные системы читаются от контроллера к полевому устройству: для пожарной сигнализации — ППКП → адресный шлейф → извещатель/модуль; для СКУД — контроллер → считыватель → замок; для СОТ — камера → коммутатор → NVR/сервер. Каждая цепочка устройств должна быть описана как одна связная запись: контроллер → кабель → полевое устройство, с сохранением адресной/канальной привязки.

## Правила

1. **Главное правило:** для каждого устройства/датчика на чертеже зафиксировать: обозначение, тип/модель, расположение, подключение (к какому контроллеру/шлейфу/коммутатору), тип кабеля.

2. **Если на чертеже несколько подсистем** (напр., АПС + СОУЭ на одном листе) — описать каждую подсистему отдельно с чёткими заголовками.

3. **Если параметр не читается** на изображении — написать "нечитаемо" вместо угадывания.

4. **Количество устройств:** всегда считать и указывать общее количество каждого типа устройств. Это критически важно для перекрёстной проверки со спецификациями.

5. **Типы кабелей важны:** для пожарной сигнализации тип кабеля (FR/FRLS/FRHF) — критический параметр. Всегда фиксировать его.

6. **Распределение адресов/зон:** если у устройств видны адреса или номера зон — зафиксировать их. Это нужно для проверки ёмкости шлейфов.

## Типичные ошибки описания (чего избегать)

X **Плохо:** "На чертеже показана система пожарной сигнализации с дымовыми извещателями и сиренами."
-> Нет конкретики, нет количеств, нет подключений

X **Плохо:** "ППКП имеет несколько шлейфов, подключённых к извещателям на разных этажах."
-> "Несколько" и "разных" бесполезны — указывайте точные числа

V **Хорошо:** "Шлейф 1 (С2000-КДЛ): 47 дымовых извещателей ДИП-34А + 6 ручных извещателей ИПР 513-3А + 4 изолятора КЗ Б2Р-КДЛ, этажи 1-5. Шлейф 2: ..."
-> Точные количества, типы, распределение по этажам
