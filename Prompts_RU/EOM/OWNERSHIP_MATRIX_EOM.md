# Матрица владения проверками — EOM

## Базовый принцип

Каждая проверка имеет **ОДНОГО** владельца. Остальные агенты **НЕ** дублируют эту проверку. Если агент обнаружил проблему вне своей зоны — он записывает её **только** в `checklist.notes`, но **не** создаёт finding.

## Типы проверок

| Тип проверки | Единственный владелец | Описание |
|---|---|---|
| **Консистентность** — буквальное расхождение одного параметра между источниками | `consistency` | Один и тот же параметр отличается в тексте, схеме, плане, спецификации или компоновке |
| **Арифметика** — пересчёт формул, сумм, коэффициентов внутри таблицы | `tables` | Ошибка обнаруживается через математический пересчёт |
| **Инженерная корректность** — адекватность решения в доменной зоне | профильный R1-агент | Решение технически неверно по нормам или инженерной практике |
| **Статус нормы** — актуальность документа, корректность ссылки на пункт | `norms` | Норма отменена/заменена, пункт указан неверно |

**Правило разрешения:** если не ясно, кому принадлежит проверка — определи **тип** по таблице выше.

## Владение проверками по агентам

### `consistency` (консистентность документации)

| Проверка | Владелец |
|----------|----------|
| Буквальное расхождение текст ↔ схема (имена, марки, параметры) | consistency |
| Буквальное расхождение схема ↔ план (наличие, расположение) | consistency |
| Буквальное расхождение схема ↔ спецификация (марка, тип, номинал, количество) | consistency |
| Буквальное расхождение план ↔ спецификация (наличие, количество) | consistency |
| Буквальное расхождение компоновка ↔ схема (количество панелей, расположение) | consistency |
| Инвентаризация листов (ведомость vs фактическое наличие) | consistency |
| Серые зоны покрытия (лист не покрыт ни одним агентом) | consistency |

**НЕ владеет:**
- штампами, шифрами, наименованиями листов, изменениями
- косметикой оформления, условными обозначениями, легендами
- арифметикой (пересчёт формул) → `tables`
- инженерной оценкой правильности решения → профильный агент
- статусом нормативных ссылок → `norms`

---

### `tables` (арифметика и внутренняя математика)

| Проверка | Владелец |
|----------|----------|
| Арифметика таблицы нагрузок (Pу, Pр, Qр, Sр, Iр) | tables |
| Итоговые суммы по ВРУ / секциям / ГРЩ | tables |
| Внутренняя согласованность коэффициентов (Kс, cosφ, Ко, Kн.макс) | tables |
| Арифметика таблицы ТТ (пересчёт формул по режимам) | tables |

**НЕ владеет:**
- полной проверкой спецификации (количество vs схема/план) → `consistency`
- нумерацией и порядком позиций в спецификации
- инженерной оценкой выбора коэффициентов → notes
- «резервные автоматы ≥10%» как универсальное требование → notes

---

### `cables` (инженерная корректность кабельных линий)

| Проверка | Владелец |
|----------|----------|
| Сечения кабелей по допустимому току (Iрасч ≤ Iдоп) | cables |
| Координация автомат ↔ кабель (Iном ≤ Iдоп, Iном ≥ Iрасч) | cables |
| Характеристики B/C/D групповых автоматов | cables |
| Отключающая способность (Ics ≥ Iкз) | cables |
| Потери напряжения (суммарные ≤ 5%) | cables |

**НЕ владеет:**
- расхождением марки кабеля между источниками (текст ↔ схема ↔ спецификация) → `consistency`
- экономической плотностью тока как finding → notes
- арифметикой таблицы нагрузок → `tables`

---

### `fire_safety` (электротехническая пожарная безопасность)

| Проверка | Владелец |
|----------|----------|
| FR/FRLS/FRHF — огнестойкость кабелей для линий СПЗ | fire_safety |
| EI45/EI60/EI150 — требуемый предел огнестойкости трасс | fire_safety |
| Разделение взаиморезервируемых линий | fire_safety |
| Кабельные проходки через противопожарные преграды | fire_safety |
| ПЭСПЗ / режим «Пожар» на схеме, питание СПЗ | fire_safety |
| Аварийное / эвакуационное освещение (указатели EXIT, автономия ≥3ч) | fire_safety |
| Дымоудаление, подпор воздуха, пуск от ОПС | fire_safety |
| Рабочее/аварийное освещение на разных лотках | fire_safety |

**НЕ владеет:**
- общим IP оборудования по условиям среды → `lighting` (светильники), `power_equipment` (силовое)
- расхождением марки кабеля между источниками → `consistency`
- категориями помещений как самостоятельный finding (только как контекст для FR/EI)

---

### `cable_routes` (конструкция кабельных трасс)

| Проверка | Владелец |
|----------|----------|
| Заполнение лотков (≤ 40%) | cable_routes |
| Совместная прокладка силовых + слаботочных | cable_routes |
| Узлы крепления (шпильки, отметки низа, консоли) | cable_routes |
| Конструкция огнезащитных коробов (внутренние размеры, толщина, вентрешётки) | cable_routes |

**НЕ владеет:**
- расхождением типоразмера лотка план ↔ спецификация → `consistency`
- расхождением длин план ↔ спецификация → `consistency`
- требуемым пределом огнестойкости EI как пожарное требование → `fire_safety`

---

### `lighting` (обычное и архитектурное освещение)

| Проверка | Владелец |
|----------|----------|
| IP светильников по условиям эксплуатации | lighting |
| Мощностной баланс освещения | lighting |
| DALI: контроллеры, БП, ≤64 на линию | lighting |
| Нормы освещённости (лк) — только при наличии расчёта/ТЗ | lighting |

**НЕ владеет:**
- аварийным / эвакуационным освещением → `fire_safety`
- расхождением «светильник на плане, но нет в спецификации» → `consistency`
- пропусками нумерации позиций в спецификации

---

### `metering` (система учёта электроэнергии)

| Проверка | Владелец |
|----------|----------|
| Выбор ТТ (Ктт, класс точности, обоснованность по рабочему/аварийному току) | metering |
| Счётчики (модель, интерфейс, совместимость с АСКУЭ) | metering |
| ИКК (наличие, схема подключения) | metering |
| Точки учёта (коммерческий / технический) | metering |

**НЕ владеет:**
- арифметикой таблицы ТТ (пересчёт формул) → `tables`
- расхождением модели/количества ТТ между схемой и спецификацией → `consistency`

---

### `automation` (логика управления)

| Проверка | Владелец |
|----------|----------|
| Логика управления (сценарии, SA/K/KM) | automation |
| Связь с АСУД (XT1/XT2, клеммы, протокол) | automation |
| Астрономические реле, таймеры | automation |
| АВР для щитов освещения | automation |

**НЕ владеет:**
- «нет задания смежному разделу» как finding → notes
- отсутствием резервного астрореле как обязательное замечание → notes

---

### `grounding` (заземление и молниезащита)

| Проверка | Владелец |
|----------|----------|
| TN-C-S, точка разделения PEN | grounding |
| ГЗШ и подключения к ней | grounding |
| Уравнивание потенциалов (ОСУП, ДСУП, КУП) | grounding |
| Молниезащита (молниеприёмники, токоотводы) | grounding |
| PE-проводники (сечения) | grounding |

**НЕ владеет:**
- ДСУП в помещениях, не охваченных данным разделом проекта

---

### `power_equipment` (силовое электрооборудование)

| Проверка | Владелец |
|----------|----------|
| Электродвигатели (тип пуска, защита) | power_equipment |
| Розеточные сети (УЗО 30мА, выделенные линии) | power_equipment |
| Кабельный обогрев (мощность, терморегулятор, УЗО) | power_equipment |
| Частотные преобразователи | power_equipment |
| Блокировки с технологией (насосы, общеобменная вентиляция) | power_equipment |

**НЕ владеет:**
- комплектацией щитов по спецификации (схема ↔ спецификация) → `consistency`
- «резервные группы» как универсальное требование → notes

---

### `outdoor_install` (наружная прокладка и земляные работы)

| Проверка | Владелец |
|----------|----------|
| Глубина заложения кабелей в траншеях | outdoor_install |
| Расстояния от коммуникаций | outdoor_install |
| Футляры под дорогами | outdoor_install |
| Объёмы земляных работ (арифметика) | outdoor_install |

**НЕ владеет:**
- «расстояния не указаны, только ссылка на ПУЭ» как finding → notes

---

### `norms` (нормативные ссылки)

| Проверка | Владелец |
|----------|----------|
| Статус нормы (действующая / отменённая / заменённая) | norms |
| Корректность номера пункта и цитаты | norms |

**НЕ владеет:**
- «в перечне нет СП 256» как самостоятельный finding → notes
- «ПУЭ без параллельной ссылки на СП» как finding → notes
- общей полнотой перечня нормативных документов как finding → notes
