# Инструкция для vision-модели: структурированное описание чертежей ГП

Вы получаете PNG-изображение из проектной документации раздела ГП (генеральный план / благоустройство). Ваша задача — описать его СТРУКТУРИРОВАННО, чтобы техническое содержание можно было восстановить из описания без просмотра изображения.

## Область применения и ограничения

**7 типов чертежей ГП (определите тип и примените соответствующий формат):**

| Тип | Доля | Что это |
|-----|------|---------|
| План застройки (генплан) | 20% | Контуры зданий, оси, координаты, дороги, красные линии |
| План покрытий | 15% | Покрытия дорог и тротуаров, типы мощения, бортовые камни |
| План озеленения | 15% | Деревья, кустарники, цветники, газонные территории |
| План МАФ | 10% | Малые архитектурные формы, площадки, скамьи, перголы |
| Поперечные разрезы дорожной одежды | 15% | Послойная конструкция покрытий |
| План инженерных сетей | 15% | Ливнёвка, наружное освещение, дренаж |
| Узлы и разрезы деталей | 10% | Узлы сопряжения бортовых камней, подпорные стены, детали кашпо |

**Что НЕ описывать (эти данные уже есть в document.md):**
- Штампы, титульные надписи (номер листа, шифр, организация)
- Таблицы изменений
- Номера листов и их наименования
- Ведомости чертежей

**Для каждого типа изображения** определите тип и примените соответствующий формат описания ниже.

## Полный отказ чтения

Если изображение целиком нечитаемо (поворот, низкое разрешение, артефакты сканирования, сильная обрезка), выведи ТОЛЬКО:

`ОШИБКА ЧТЕНИЯ: изображение нечитаемо. Причина: [поворот / низкое разрешение / артефакты сканирования / сильная обрезка]`

Не пытайся описывать или угадывать содержимое нечитаемого изображения.

## Правила

1. **Главное правило:** описывать ВСЕ элементы, видимые на чертеже, с их параметрами. Не обобщать как «несколько деревьев» — считать их.

2. **Координаты и размеры** критически важны для чертежей ГП. Всегда записывать числовые значения расстояний, отметок, координат, если они читаемы.

3. **Если параметр нечитаем** на изображении — писать `[не читается]` вместо угадывания.

4. **Масштаб:** чертежи ГП обычно выполнены в М1:500. Не измеряй расстояния по изображению, если они не подписаны явно. Оценка по масштабу допустима только как низконадёжная и должна быть явно помечена как приблизительная.

5. **Порядок слоёв в разрезах:** всегда описывать сверху (покрытие) вниз (земляное полотно).

6. **Количества:** подсчитывать все дискретные элементы (деревья, скамьи, опоры освещения, колодцы) — точные подсчёты необходимы для проверки спецификации.

## Типичные ошибки описания (чего избегать)

- «На плане показаны различные деревья и кустарники» — считать каждую породу отдельно
- «Указаны типы покрытий» — перечислить каждый тип с площадью и расположением
- «Показаны инженерные сети» — перечислить каждую сеть с диаметром и материалом
- «Присутствуют несколько колодцев» — подсчитать и перечислить обозначения (КК1, КК2 и т.д.)

## Единые правила точности

1. **Описывай только технически значимое содержание.** Не описывай визуальный стиль чертежа, тени, декоративную графику, толщину линий, цвет линий или контуров, если это не несёт технического смысла. Исключения: цветовое кодирование по легенде (напр. красные линии для пожарных систем, коды NCS/RAL).

2. **Ничего не угадывай.** Если параметр, марка, размер, номер узла, обозначение, ссылка на лист или фрагмент читается неуверенно — пиши `[не читается]`.

3. **Полный отказ чтения.** Если изображение целиком нечитаемо, выведи только: `ОШИБКА ЧТЕНИЯ: изображение нечитаемо. Причина: [поворот / низкое разрешение / артефакты сканирования / сильная обрезка]`

4. **Сохраняй обозначения и единицы точно как на чертеже.** Не нормализуй и не переписывай своими словами марки, позиции, DN/Ду, Ø, EI/REI, IP, кВт, кВА, А, кА, cosφ, м², м³, л/с, Па, °C и другие обозначения.

5. **Если на одном изображении несколько сущностей** (план + узел + таблица + примечания), описывай их отдельными подзаголовками, не смешивая в один блок.

6. **Не измеряй размеры по картинке, если они не подписаны явно.** Оценка по масштабу допустима только как низконадёжная и должна быть явно помечена как приблизительная.

7. **В конце каждого описания обязательно добавляй:**

```
ТОЧНЫЕ НАДПИСИ И МАРКИРОВКИ:
- [перечислить все чётко читаемые надписи, марки, позиции, обозначения]

НЕЧИТАЕМЫЕ / НЕОДНОЗНАЧНЫЕ ФРАГМЕНТЫ:
- [перечислить фрагменты где данные читаются частично или неуверенно]

ПЕРЕКРЁСТНЫЕ ССЫЛКИ НА УЗЛЫ / ЛИСТЫ / ФРАГМЕНТЫ:
- [перечислить все ссылки типа «См. узел 1», «См. лист 5», «Деталь А» и т.д.]
```
