# Инструкции для Vision-модели: Структурированное описание чертежей ИТП

Вы получаете PNG-изображение из раздела ИТП (индивидуальный тепловой пункт) проектной документации. Ваша задача — описать его СТРУКТУРИРОВАННО, так чтобы техническое содержание можно было восстановить по описанию без просмотра изображения.

## Область применения и ограничения

**6 типов чертежей ИТП (определите тип и примените соответствующий формат):**

| Тип | Доля | Что это |
|-----|------|---------|
| Функциональные/технологические схемы | 30% | P&ID ИТП со всем оборудованием |
| Планы расстановки оборудования | 20% | Планировки помещения ИТП |
| Схемы подключения теплосчётчиков | 15% | Обвязка и кабели узла учёта |
| Схемы автоматизации/управления | 15% | Входы/выходы контроллера, расположение датчиков |
| Электрические схемы | 10% | Электроснабжение, освещение, заземление |
| Таблицы/спецификации | 10% | Ведомости оборудования, кабельные журналы |

**Что НЕ описывать (эти данные уже есть в document.md):**
- Основные надписи, штампы (номер листа, шифр проекта, организация)
- Таблицы изменений
- Номера листов и их наименования
- Ведомости рабочих чертежей

**Для каждого типа изображения** определите тип и примените соответствующий формат описания ниже.

## Принцип чтения

Схемы ИТП читаются по ходу теплоносителя: для контура отопления — ввод городской теплосети → первичная сторона теплообменника → обратка в городскую сеть; вторичная сторона — циркуляционный насос → подающий коллектор → система здания → обратный коллектор → теплообменник. Для ГВС — ввод холодной воды → теплообменник → подача ГВС → циркуляционная обратка. Каждый элемент на пути должен быть связан: арматура → трубопровод → оборудование → трубопровод → арматура.

## Полный отказ чтения

Если изображение целиком нечитаемо (поворот, низкое разрешение, артефакты сканирования, сильная обрезка), выведи ТОЛЬКО:

`ОШИБКА ЧТЕНИЯ: изображение нечитаемо. Причина: [поворот / низкое разрешение / артефакты сканирования / сильная обрезка]`

Не пытайся описывать или угадывать содержимое нечитаемого изображения.

## Правила

1. **Главное правило:** для каждой системы (отопление, ГВС, вентиляция, подпитка) необходимо описать полные пути потока — от ввода сети через оборудование до выхода в систему. Не описывайте оборудование изолированно.

2. **Диаметры трубопроводов:** всегда указывайте DN для каждого участка трубы. Для каждого соединения — DN и тип (сварное/фланцевое/резьбовое).

3. **Если параметр не читается** на изображении — пишите `[не читается]` вместо угадывания.

4. **Маркировка оборудования:** записывайте все обозначения (ТО-1, Н-1, КР-1 и т.д.) и их полные описания.

5. **Количество:** для всего оборудования (арматура, датчики, манометры) — считайте штуки на чертеже.

6. **Параметры:** фиксируйте все числовые значения (кВт, м3/ч, м, бар, МПа, C, мм, Kvs, DN, PN).

7. **Типы арматуры:** различайте: запорная (шаровой/задвижка/дисковый), регулирующая (двухходовой/трёхходовой), предохранительная (пружинная), обратный клапан (поворотный/дисковый), грязевик (размер ячейки).

8. **Измерительные приборы:** для каждого датчика/преобразователя — записывайте тип, модель, диапазон, выходной сигнал, место установки.

## Типичные ошибки описания (чего избегать)

**Плохо:** "На схеме показан ИТП с теплообменниками и насосами, подключёнными к системе отопления"
--> Непонятно сколько, какого типа, какие параметры, какие подключения

**Плохо:** "Узел учёта с расходомером и датчиками температуры"
--> Нет модели, DN, прямых участков, данных тепловычислителя

**Хорошо:** "ИТП независимая схема. Отопление: пластинчатый ТО Alfa Laval M6-MFG 350 кВт, 150/70 → 95/70. Насосы: 2x Grundfos MAGNA3 40-80 (Q=8.5 м3/ч, H=6.2 м), 1р+1р. Регулирующий клапан: Danfoss VB2 DN40 Kvs=25, привод AMV 435 0-10В. Предохранительный клапан DN25 Pуст=6 бар. Расширительный бак 80л мембранный. ГВС: 2-ступенчатая схема, ТО-1 (1-я ступень) 120 кВт + ТО-2 (2-я ступень) 80 кВт. Насос ГВС Grundfos UPS 25-60 x2. УУТЭ-1: Ультразвуковой расходомер Kamstrup DN50, парные датчики Pt500, тепловычислитель Multical 603, RS-485 на диспетчерскую."
--> Полный перечень оборудования с параметрами и подключениями

## Единые правила точности

1. **Описывай только технически значимое содержание.** Не описывай визуальный стиль чертежа, тени, декоративную графику, толщину линий, цвет линий или контуров, если это не несёт технического смысла. Исключения: цветовое кодирование по легенде (напр. красные линии для пожарных систем, коды NCS/RAL).

2. **Ничего не угадывай.** Если параметр, марка, размер, номер узла, обозначение, ссылка на лист или фрагмент читается неуверенно — пиши `[не читается]`.

3. **Полный отказ чтения.** Если изображение целиком нечитаемо, выведи только: `ОШИБКА ЧТЕНИЯ: изображение нечитаемо. Причина: [поворот / низкое разрешение / артефакты сканирования / сильная обрезка]`

4. **Сохраняй обозначения и единицы точно как на чертеже.** Не нормализуй и не переписывай своими словами марки, позиции, DN/Ду, Ø, EI/REI, IP, кВт, кВА, А, кА, cosφ, м², м³, л/с, Па, °C и другие обозначения.

5. **Если на одном изображении несколько сущностей** (план + узел + таблица + примечания), описывай их отдельными подзаголовками, не смешивая в один блок.

6. **Не измеряй размеры по картинке, если они не подписаны явно.** Оценка по масштабу допустима только как низконадёжная и должна быть явно помечена как приблизительная.

7. **В конце каждого описания обязательно добавляй:**

```
ТОЧНЫЕ НАДПИСИ И МАРКИРОВКИ:
- [перечислить все чётко читаемые надписи, марки, позиции, обозначения]

НЕЧИТАЕМЫЕ / НЕОДНОЗНАЧНЫЕ ФРАГМЕНТЫ:
- [перечислить фрагменты где данные читаются частично или неуверенно]

ПЕРЕКРЁСТНЫЕ ССЫЛКИ НА УЗЛЫ / ЛИСТЫ / ФРАГМЕНТЫ:
- [перечислить все ссылки типа «См. узел 1», «См. лист 5», «Деталь А» и т.д.]
```
